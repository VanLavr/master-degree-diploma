# Функциональные требования к кнопке «Валидировать»

## 1. Назначение функции

Функция «Валидировать» предназначена для автоматической проверки корректности спроектированного технологического процесса химико-термической обработки изделия до запуска моделирования и формирования отчёта.

Валидация должна анализировать:

* наличие выбранной CAD-модели;
* корректность твердотельной геометрии;
* наличие и допустимость материала;
* полноту описания технологического процесса;
* корректность последовательности этапов ХТО;
* допустимость параметров каждого этапа;
* соответствие параметров процесса выбранному материалу;
* потенциальные геометрические риски изделия при ХТО;
* соответствие процесса правилам, заданным в `rules.yaml`.

Функция не должна выполнять полноценное физическое моделирование. Её задача — определить, можно ли считать введённый техпроцесс корректным и пригодным для дальнейшего расчёта.

---

## 2. Входные данные для валидации

### FR-VAL-01. Получение активной модели

Плагин должен определять активную или выбранную пользователем твердотельную модель FreeCAD.

Если модель не выбрана или документ пустой, валидация должна завершаться ошибкой:

> Не выбрана модель изделия для проверки.

---

### FR-VAL-02. Проверка корректности геометрии

Плагин должен проверять, является ли выбранный объект корректным твердотельным телом.

Проверяются:

* наличие `Shape`;
* тип объекта: `Solid`, `Body`, `Part::Feature` или совместимый объект;
* замкнутость тела;
* валидность формы;
* отсутствие пустой геометрии;
* наличие граней, рёбер и объёма.

Пример ошибки:

> Геометрия модели некорректна: тело не является замкнутым solid-объектом.

---

### FR-VAL-03. Извлечение геометрических параметров модели

Плагин должен автоматически извлекать из CAD-модели параметры, значимые для ХТО.

К базовым параметрам относятся:

* габаритные размеры изделия;
* объём;
* площадь поверхности;
* отношение площади поверхности к объёму;
* количество граней;
* количество рёбер;
* количество цилиндрических поверхностей;
* наличие отверстий;
* наличие пазов и канавок;
* наличие острых кромок;
* минимальный радиус скругления;
* минимальная локальная толщина стенки, если её возможно определить;
* наличие тонкостенных участков;
* наличие длинных тонких элементов;
* наличие массивных зон;
* наличие резких переходов сечений.

Часть параметров может быть вычисляемой, а не напрямую хранящейся во FreeCAD.

---

### FR-VAL-04. Извлечение материала изделия

Плагин должен определять материал изделия одним из следующих способов:

1. из свойства модели FreeCAD;
2. из данных, указанных пользователем в интерфейсе плагина;
3. из значения по умолчанию, заданного в `rules.yaml`.

Приоритет источников материала должен настраиваться в `rules.yaml`.

Пример параметра:

```yaml
material_policy:
  source_priority:
    - process_form
    - freecad_object
    - default
  default_material: "20Х"
  allow_default_material: true
```

---

### FR-VAL-05. Проверка наличия технологического процесса

Плагин должен проверять, создан ли предварительный план ХТО.

Если план отсутствует, должно выводиться сообщение:

> Не создан технологический процесс ХТО. Добавьте этапы обработки перед валидацией.

---

## 3. Проверка этапов технологического процесса

### FR-VAL-06. Проверка обязательных этапов

Плагин должен проверять наличие обязательных этапов процесса.

Для цементации минимальный процесс может включать:

* нагрев;
* цементацию;
* диффузионную выдержку;
* закалку;
* отпуск.

Состав обязательных этапов должен задаваться в `rules.yaml`.

Пример:

```yaml
process_rules:
  carburizing:
    required_stages:
      - preheating
      - carburizing
      - quenching
      - tempering
```

---

### FR-VAL-07. Проверка последовательности этапов

Плагин должен проверять, что этапы расположены в допустимом порядке.

Например, закалка не может идти до цементации, а отпуск не может выполняться до закалки.

Пример ошибки:

> Нарушена последовательность этапов: этап «Отпуск» расположен перед этапом «Закалка».

---

### FR-VAL-08. Проверка обязательных параметров этапов

Для каждого этапа плагин должен проверять наличие обязательных параметров.

Например, для цементации:

* температура;
* длительность;
* углеродный потенциал;
* среда обработки.

Для закалки:

* температура нагрева;
* закалочная среда;
* время выдержки;
* температура среды, если указана.

Для отпуска:

* температура;
* длительность;
* количество циклов, если используется повторный отпуск.

---

### FR-VAL-09. Проверка диапазонов параметров

Плагин должен проверять, что параметры каждого этапа находятся в допустимых диапазонах.

Примеры проверок:

* температура цементации не ниже минимальной;
* температура цементации не выше максимальной;
* длительность выдержки положительная;
* углеродный потенциал находится в допустимом диапазоне;
* температура отпуска соответствует материалу;
* время отпуска не меньше минимального;
* среда закалки допустима для выбранной стали.

Диапазоны должны задаваться в `rules.yaml`.

---

### FR-VAL-10. Проверка совместимости материала и процесса

Плагин должен проверять, допускается ли выбранный процесс ХТО для выбранного материала.

Например:

* низкоуглеродистая легированная сталь может быть допустима для цементации;
* материал с высоким содержанием углерода может быть помечен как нежелательный для цементации;
* материал без заданных свойств должен вызывать предупреждение или ошибку в зависимости от настроек.

Пример ошибки:

> Материал «У8» не рекомендуется для цементации: содержание углерода превышает допустимое значение.

---

## 4. Геометрическая валидация изделия

### FR-VAL-11. Проверка тонких стенок

Плагин должен определять тонкостенные зоны изделия и сравнивать их с минимально допустимой толщиной из `rules.yaml`.

Пример предупреждения:

> Обнаружена тонкая стенка 1.2 мм. Возможен перегрев или коробление при ХТО.

---

### FR-VAL-12. Проверка острых кромок

Плагин должен анализировать рёбра и радиусы скруглений.

Если радиус кромки меньше допустимого значения, плагин должен выдавать предупреждение.

Пример:

> Обнаружены острые кромки с радиусом менее 0.3 мм. Возможна локальная концентрация напряжений и неравномерное насыщение поверхности.

---

### FR-VAL-13. Проверка отверстий и внутренних полостей

Плагин должен определять наличие отверстий, глухих отверстий и внутренних полостей, если это возможно по геометрии.

Для таких зон должны формироваться предупреждения о возможной неравномерности прогрева, насыщения углеродом или охлаждения.

Пример:

> Обнаружено отверстие малого диаметра. Возможна неравномерная цементация внутренней поверхности.

---

### FR-VAL-14. Проверка резких переходов сечений

Плагин должен выявлять участки с резким изменением толщины или массивности.

Такие зоны должны помечаться как потенциально опасные с точки зрения термических напряжений и коробления.

---

### FR-VAL-15. Проверка отношения площади поверхности к объёму

Плагин должен вычислять отношение площади поверхности к объёму изделия.

Если отношение превышает заданный порог, изделие может считаться склонным к быстрому нагреву и охлаждению.

Если отношение слишком низкое, изделие может считаться массивным, что повышает риск недостаточного прогрева сердцевины.

---

### FR-VAL-16. Проверка длинных тонких элементов

Плагин должен определять вытянутые элементы с большим отношением длины к толщине.

Такие элементы должны помечаться как зоны риска коробления при закалке.

---

## 5. Валидация требований к результату ХТО

### FR-VAL-17. Проверка целевой глубины слоя

Если пользователь указал требуемую глубину цементированного слоя, плагин должен проверить, что она находится в допустимом диапазоне для выбранного материала и процесса.

Пример:

> Требуемая глубина слоя 2.5 мм превышает рекомендуемый диапазон для выбранного процесса.

---

### FR-VAL-18. Проверка целевой твёрдости

Если пользователь указал требуемую поверхностную твёрдость или твёрдость сердцевины, плагин должен проверить, достижимы ли эти значения для выбранного материала.

Пример:

> Требуемая поверхностная твёрдость 65 HRC может быть недостижима для выбранного материала.

---

### FR-VAL-19. Проверка согласованности параметров

Плагин должен выявлять противоречия между параметрами процесса.

Примеры:

* высокая требуемая глубина слоя при слишком коротком времени цементации;
* низкая температура при большой требуемой глубине слоя;
* выбранная среда закалки не соответствует требуемой твёрдости;
* материал не содержит данных, необходимых для оценки твёрдости.

---

## 6. Работа с rules.yaml

### FR-VAL-20. Загрузка правил из rules.yaml

Плагин должен загружать файл `rules.yaml` при запуске валидации.

Если файл отсутствует, повреждён или содержит некорректную структуру, должна выводиться ошибка.

Пример:

> Не удалось загрузить rules.yaml: отсутствует раздел material_policy.

---

### FR-VAL-21. Поддержка включения и отключения отдельных правил

Каждое правило должно иметь возможность включения или отключения через `rules.yaml`.

Пример:

```yaml
geometry_rules:
  check_thin_walls:
    enabled: true
    min_thickness_mm: 2.0

  check_sharp_edges:
    enabled: true
    min_radius_mm: 0.3
```

---

### FR-VAL-22. Поддержка уровней строгости

Плагин должен поддерживать разные уровни строгости валидации:

* `soft` — большинство проблем считаются предупреждениями;
* `normal` — часть нарушений блокирует дальнейшее моделирование;
* `strict` — любое серьёзное отклонение считается ошибкой.

Пример:

```yaml
validation:
  strictness: "normal"
```

---

### FR-VAL-23. Настройка источников данных

В `rules.yaml` должны задаваться правила выбора источника данных.

Например:

* использовать материал из модели;
* использовать материал из формы процесса;
* использовать материал по умолчанию;
* разрешать или запрещать значения по умолчанию;
* разрешать запуск валидации при неполных данных.

---

### FR-VAL-24. Настройка сообщений и рекомендаций

Для каждого правила в `rules.yaml` может быть задан текст сообщения и рекомендация по исправлению.

Пример:

```yaml
messages:
  thin_wall:
    warning: "Обнаружена тонкая стенка: возможен риск коробления."
    recommendation: "Увеличьте толщину стенки или скорректируйте режим нагрева и охлаждения."
```

---

## 7. Результат валидации

### FR-VAL-25. Формирование итогового статуса

После завершения проверки плагин должен сформировать один из итоговых статусов:

* `VALID` — процесс ХТО корректен;
* `VALID_WITH_WARNINGS` — процесс может быть использован, но содержит предупреждения;
* `INVALID` — процесс содержит ошибки, которые требуют исправления;
* `VALIDATION_FAILED` — валидация не выполнена из-за ошибки системы, например отсутствия правил или некорректной модели.

---

### FR-VAL-26. Формирование списка сообщений

Результат валидации должен содержать список сообщений.

Каждое сообщение должно включать:

* код правила;
* уровень серьёзности;
* краткое описание проблемы;
* причину возникновения;
* рекомендацию по исправлению;
* ссылку на этап процесса или элемент модели, если применимо.

Пример структуры сообщения:

```text
[HTO-GEO-001] WARNING
Проблема: обнаружена тонкая стенка 1.2 мм.
Причина: значение меньше допустимого порога 2.0 мм.
Рекомендация: увеличить толщину стенки или снизить интенсивность охлаждения.
Объект: Body.Face12
```

---

### FR-VAL-27. Отображение результата пользователю

После нажатия кнопки «Валидировать» должно открываться окно результата проверки.

Окно должно содержать:

* общий статус;
* количество ошибок;
* количество предупреждений;
* количество информационных сообщений;
* таблицу найденных проблем;
* рекомендации по исправлению;

---

### FR-VAL-28. Блокировка моделирования при критических ошибках

Если результат валидации имеет статус `INVALID`, плагин должен запрещать запуск моделирования и выводить предупреждение при нажатии на кнопку запуска моделирования.

Пример:

> Процесс содержит критические ошибки. Запуск моделирования невозможен до исправления параметров.

---

## 8. Требования к интеграции с FreeCAD

### FR-VAL-29. Неразрушающая работа с моделью

Валидация не должна изменять геометрию исходной детали.

Допускаются только:

* создание вспомогательных объектов;
* создание объекта результата валидации;
* запись пользовательских свойств плагина.

---

### FR-VAL-30. Работа с импортированными моделями

Плагин должен работать не только с параметрическими моделями FreeCAD, но и с импортированными STEP/IGES-моделями, если они представлены корректным твердотельным телом.

При этом плагин не должен зависеть от истории построения модели.

---

### FR-VAL-31. Обработка ошибок геометрического анализа

Если отдельный геометрический параметр невозможно вычислить, плагин не должен аварийно завершать работу.

Вместо этого должно выводиться предупреждение:

> Не удалось определить минимальную толщину стенки. Проверка тонкостенных участков пропущена.

---

## 9. Нефункциональные требования

### NFR-VAL-01. Объяснимость результата

Каждое найденное нарушение должно сопровождаться понятным объяснением и рекомендацией.

---

### NFR-VAL-02. Настраиваемость

Основные правила валидации должны изменяться через `rules.yaml` без изменения исходного кода плагина.

---

### NFR-VAL-03. Повторяемость

При одинаковой модели, одинаковом техпроцессе и одинаковом файле правил результат валидации должен быть одинаковым.

---

### NFR-VAL-04. Отказоустойчивость

Ошибки чтения правил, отсутствующие свойства модели и невозможность вычисления отдельных параметров не должны приводить к падению FreeCAD.

---

### NFR-VAL-05. Производительность

Валидация типовой детали должна выполняться за приемлемое время и не блокировать интерфейс FreeCAD на длительный период.

Для сложных моделей допускается отображение индикатора выполнения.

---

## 10. Рекомендуемая логика работы кнопки

Алгоритм работы функции «Валидировать»:

1. Получить активную модель.
2. Проверить корректность геометрии.
3. Загрузить `rules.yaml`.
4. Получить материал изделия.
5. Получить список этапов ХТО.
6. Проверить полноту процесса.
7. Проверить последовательность этапов.
8. Проверить параметры каждого этапа.
9. Проверить совместимость материала и процесса.
10. Извлечь геометрические параметры модели.
11. Выполнить геометрические проверки.
12. Сформировать список ошибок и предупреждений.
13. Определить итоговый статус.
14. Отобразить результат пользователю.
15. Подсветить проблемные зоны модели.
16. Сохранить результат валидации для отчёта и моделирования.

---

## 11. Пример результата успешной валидации

```text
Результат валидации: процесс ХТО валиден.

Ошибки: 0
Предупреждения: 0

Материал: 20Х
Процесс: цементация + закалка + отпуск
Модель: Body
Геометрия: корректная
Обязательные этапы: присутствуют
Последовательность этапов: корректная
Параметры этапов: находятся в допустимых диапазонах
Геометрические риски: не обнаружены
```

---

## 12. Пример результата валидации с предупреждениями

```text
Результат валидации: процесс ХТО содержит предупреждения.

Ошибки: 0
Предупреждения: 3

1. [HTO-GEO-001] Обнаружена тонкая стенка 1.2 мм.
   Рекомендация: увеличить толщину стенки или использовать менее интенсивное охлаждение.

2. [HTO-MAT-002] Материал модели не задан. Использован материал по умолчанию: 20Х.
   Рекомендация: явно указать материал изделия.

3. [HTO-PROC-004] Время цементации близко к минимально допустимому.
   Рекомендация: проверить требуемую глубину цементированного слоя.
```

---

## 13. Пример результата неуспешной валидации

```text
Результат валидации: процесс ХТО невалиден.
```
