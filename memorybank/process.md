# Функциональные требования к кнопке «Создать процесс»

## 1. Назначение функции

Функция «Создать процесс» предназначена для создания в текущем документе FreeCAD объекта технологического процесса химико-термической обработки изделия.

Созданный процесс должен быть связан с текущей CAD-моделью.

Функция не выполняет расчёт, моделирование или проверку корректности параметров ХТО. Её задача — создать исходную структуру процесса, которую пользователь затем может редактировать.

---

## 2. Общая логика работы функции

После нажатия кнопки «Создать процесс» плагин должен:

1. определить активный документ FreeCAD;
2. определить выбранную или активную модель изделия;
3. проверить наличие уже созданного процесса ХТО;
4. если процесс уже существует — уведомить пользователя;
5. если процесс отсутствует — создать новый объект процесса в дереве модели;
6. добавить в процесс этапы по умолчанию;
7. заполнить параметры процесса начальными значениями;
8. связать процесс с выбранной CAD-моделью;
9. отобразить созданный процесс в дереве проекта.

---

## 3. Проверка исходных условий

### FR-CP-01. Проверка активного документа

Плагин должен проверять наличие открытого активного документа FreeCAD.

Если активный документ отсутствует, процесс не должен создаваться.

Сообщение пользователю:

```text
Невозможно создать процесс ХТО: отсутствует активный документ FreeCAD.
```

---

### FR-CP-02. Проверка наличия модели изделия

Плагин должен проверять наличие выбранной или активной CAD-модели изделия.

Если модель не выбрана, плагин должен вывести сообщение:

```text
Невозможно создать процесс ХТО: не выбрана модель изделия.
```

---

### FR-CP-03. Проверка типа объекта модели

Плагин должен проверять, что выбранный объект может быть использован как изделие для процесса ХТО.

Допустимые типы объектов:

* `Part::Feature`;
* `PartDesign::Body`;
* объект, содержащий `Shape`;
* импортированное твердотельное тело STEP/IGES, представленное как корректный объект FreeCAD.

Если выбранный объект не содержит геометрию, процесс не должен создаваться.

---

## 4. Проверка существующего процесса

### FR-CP-04. Проверка наличия процесса ХТО в документе

Перед созданием нового процесса плагин должен проверять, существует ли в текущем документе объект процесса ХТО.

Если процесс уже существует, плагин не должен автоматически создавать дубликат.

Сообщение пользователю:

```text
Процесс ХТО для текущей модели уже существует.
```

---

### FR-CP-05. Проверка процесса, связанного с выбранной моделью

Если в документе может существовать несколько моделей, плагин должен проверять наличие процесса именно для выбранной модели.

Если процесс существует для другой модели, это не должно блокировать создание процесса для текущей модели.

Рекомендуемая логика:

* один процесс ХТО на одну CAD-модель;
* несколько процессов в одном документе допускаются только при наличии нескольких моделей;
* дублирование процесса для одной и той же модели не допускается.

---

### FR-CP-06. Поведение при найденном существующем процессе

Если процесс уже существует, плагин должен предложить пользователю одно из действий:

* открыть существующий процесс;
* отменить создание нового процесса.

Для первой версии допускается более простое поведение: показать уведомление и выбрать существующий процесс в дереве модели.

---

## 5. Создание объекта процесса

### FR-CP-07. Создание объекта процесса в дереве FreeCAD

Если процесс ХТО отсутствует, плагин должен создать новый объект процесса в дереве текущего документа FreeCAD.

Рекомендуемое имя объекта:

```text
CHT_Process
```

или

```text
HTO_Process
```

Отображаемое имя:

```text
Процесс ХТО
```

---

### FR-CP-08. Связь процесса с моделью изделия

Созданный процесс должен содержать ссылку на модель изделия, для которой он был создан.

Минимально процесс должен хранить:

* имя связанной модели;
* внутренний идентификатор объекта FreeCAD;
* ссылку на объект модели, если это поддерживается выбранной архитектурой плагина.

Пример логической структуры:

```text
Process.TargetObject = Body
Process.TargetObjectName = "Body"
```

---

### FR-CP-09. Создание структуры процесса

Созданный процесс должен содержать список этапов ХТО.

Этапы должны отображаться в дереве модели как дочерние элементы процесса или храниться во внутренней структуре данных процесса с отображением в панели плагина.

Рекомендуемая структура:

```text
Процесс ХТО
  ├── Нагрев
  ├── Цементация
  ├── Закалка
  └── Отпуск
```

---

## 6. Этапы по умолчанию

### FR-CP-10. Добавление этапа «Нагрев»

При создании процесса плагин должен добавить этап «Нагрев».

Этап должен содержать начальные параметры:

* температура, °C;
* длительность, мин;
* среда нагрева, если используется;
* скорость нагрева, если поддерживается плагином.

---

### FR-CP-11. Добавление этапа «Цементация»

При создании процесса плагин должен добавить этап «Цементация».

Этап должен содержать начальные параметры:

* температура цементации, °C;
* длительность цементации, мин;
* углеродный потенциал;
* технологическая среда;
* целевая глубина цементированного слоя, м.

---

### FR-CP-12. Добавление этапа «Закалка»

При создании процесса плагин должен добавить этап «Закалка».

Этап должен содержать начальные параметры:

* температура перед закалкой, °C;
* длительность выдержки, мин;
* закалочная среда;
* температура закалочной среды, если используется;
* интенсивность охлаждения, если поддерживается плагином.

---

### FR-CP-13. Добавление этапа «Отпуск»

При создании процесса плагин должен добавить этап «Отпуск».

Этап должен содержать начальные параметры:

* температура отпуска, °C;
* длительность отпуска, мин;
* количество циклов отпуска, если поддерживается плагином.

---

### FR-CP-14. Не добавлять этап «Диффузия» по умолчанию

Этап «Диффузия» является опциональным и не должен добавляться в процесс по умолчанию.

Пользователь должен иметь возможность добавить его позднее через функцию «Добавить этап».

---

### FR-CP-15. Порядок этапов по умолчанию

Этапы по умолчанию должны создаваться в следующем порядке:

```text
Нагрев → Цементация → Закалка → Отпуск
```

Этот порядок должен использоваться как базовый технологический маршрут для процесса цементации.

---

## 7. Начальные параметры процесса

### FR-CP-16. Заполнение параметров значениями по умолчанию

При создании процесса плагин должен заполнить параметры этапов начальными значениями.

Значения по умолчанию могут храниться:

* в коде плагина;
* в конфигурационном файле;
* в `rules.yaml`, если предполагается единый источник правил и настроек.

Рекомендуется хранить значения по умолчанию в `rules.yaml`.

---

### FR-CP-17. Настройка значений по умолчанию через rules.yaml

Плагин должен позволять задавать значения этапов по умолчанию через `rules.yaml`.

Пример:

```yaml
default_process:
  type: "carburizing"
  stages:
    preheating:
      temperature_c: 850
      duration_min: 40

    carburizing:
      temperature_c: 930
      duration_min: 240
      carbon_potential: 1.1
      target_case_depth_m: 0.0008

    quenching:
      temperature_c: 830
      duration_min: 20
      medium: "oil"

    tempering:
      temperature_c: 180
      duration_min: 90
      cycles: 1
```

---

## 8. Отображение процесса в интерфейсе

### FR-CP-18. Отображение процесса в дереве модели

Созданный процесс должен отображаться в дереве текущего документа FreeCAD.

Пользователь должен видеть, что процесс связан с текущим проектом и доступен для последующего редактирования.

---

### FR-CP-19. Отображение сообщения об успешном создании

После создания процесса плагин должен вывести сообщение:

```text
Процесс ХТО успешно создан.
```

Дополнительно можно указать связанную модель:

```text
Процесс ХТО успешно создан для модели Body.
```

---

## 9. Сохранение данных процесса

### FR-CP-20. Сохранение процесса в документе FreeCAD

Созданный процесс должен сохраняться вместе с документом FreeCAD.

После закрытия и повторного открытия документа пользователь должен иметь возможность продолжить работу с ранее созданным процессом.

---

### FR-CP-21. Сохранение параметров этапов

Параметры этапов должны сохраняться как свойства объекта процесса, свойства дочерних объектов этапов или во внутренней сериализуемой структуре плагина.

Данные должны быть доступны для:

* редактирования;
* валидации;
* моделирования;
* формирования отчёта.

---

### FR-CP-22. Идентификация процесса

Каждый процесс должен иметь уникальный внутренний идентификатор.

Идентификатор нужен для связи:

* процесса;
* модели;
* результатов валидации;
* результатов моделирования;
* отчёта.

---

## 10. Обработка ошибок

### FR-CP-23. Ошибка создания объекта процесса

Если объект процесса не удалось создать, плагин должен вывести сообщение об ошибке.

Пример:

```text
Не удалось создать процесс ХТО: ошибка создания объекта FreeCAD.
```

---

### FR-CP-24. Ошибка загрузки значений по умолчанию

Если значения по умолчанию не удалось загрузить из конфигурации, плагин должен использовать встроенные базовые значения или вывести предупреждение.

Пример:

```text
Не удалось загрузить значения процесса по умолчанию из rules.yaml. Использованы встроенные значения.
```

---

### FR-CP-25. Отмена операции без изменения модели

Если создание процесса невозможно или пользователь отменяет операцию, документ FreeCAD не должен изменяться частично.

Не должно оставаться неполных объектов процесса или этапов.

---

# Нефункциональные требования к кнопке «Создать процесс»

## NFR-CP-01. Реализуемость в рамках FreeCAD Python Workbench

Функция должна быть реализуема средствами FreeCAD Python API и PySide без необходимости изменения исходного кода FreeCAD.

---

## NFR-CP-02. Неразрушающий характер работы

Создание процесса не должно изменять геометрию CAD-модели.

Допускается только создание дополнительных объектов и свойств, относящихся к технологическому процессу ХТО.

---

## NFR-CP-03. Повторяемость поведения

При одинаковой модели и одинаковой конфигурации плагин должен создавать одинаковую структуру процесса с одинаковыми начальными параметрами.

---

## NFR-CP-04. Защита от дублирования

Плагин не должен создавать несколько одинаковых процессов ХТО для одной и той же модели без явного действия пользователя.

---

## NFR-CP-05. Читаемость структуры проекта

Созданный процесс и его этапы должны иметь понятные имена в дереве FreeCAD.

Пользователь должен понимать назначение созданных объектов без анализа внутренней структуры плагина.

---

## NFR-CP-06. Сохраняемость данных

Все данные созданного процесса должны сохраняться в документе FreeCAD и восстанавливаться после повторного открытия файла.

---

## NFR-CP-07. Расширяемость

Архитектура процесса должна позволять в дальнейшем добавлять новые этапы ХТО без переработки основной логики.

Например:

* диффузия;
* предварительный подогрев;
* охлаждение;
* нормализация;
* повторный отпуск;
* очистка поверхности.

---

## NFR-CP-08. Совместимость с последующими функциями

Созданный процесс должен иметь структуру, пригодную для дальнейшей работы функций:

* «Добавить этап»;
* «Валидировать»;
* «Запустить моделирование»;
* «Отчёт».

---

## NFR-CP-09. Отказоустойчивость

Ошибка на любом этапе создания процесса не должна приводить к аварийному завершению FreeCAD.

Плагин должен корректно обработать ошибку и сообщить пользователю причину.

---

## NFR-CP-10. Минимальная зависимость от истории построения модели

Создание процесса должно работать как для параметрических моделей FreeCAD, так и для импортированных твердотельных моделей, если они представлены корректным объектом с геометрией.

---

## NFR-CP-11. Производительность

Создание процесса должно выполняться быстро и не должно приводить к заметному зависанию интерфейса FreeCAD.

---

## NFR-CP-12. Единообразие единиц измерения

Начальные параметры процесса должны использовать единицы измерения, согласованные с остальными разделами плагина.

Рекомендуемые единицы:

* температура — °C;
* длительность — мин/час;
* глубина слоя — м с возможным отображением в мм;
* твёрдость — HRC;
* риск — %.

---

# Минимальный сценарий работы кнопки

```text
Пользователь создаёт твердотельную модель.
Пользователь переключается в CHT Workbench.
Пользователь нажимает «Создать процесс».
Плагин проверяет наличие активного документа.
Плагин проверяет наличие выбранной модели.
Плагин проверяет, существует ли процесс ХТО для этой модели.
Если процесс уже существует — плагин уведомляет пользователя.
Если процесса нет — плагин создаёт объект «Процесс ХТО».
В процесс добавляются этапы: Нагрев, Цементация, Закалка, Отпуск.
Этап «Диффузия» по умолчанию не добавляется.
Плагин связывает процесс с моделью.
Плагин сохраняет процесс в дереве документа.
Плагин сообщает об успешном создании.
```
