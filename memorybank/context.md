У меня есть проект - плагин для химико-термической обработки (далее - ХТО) стали.

---

Суть, идея этого плагина в следующем:

- Пользователь может спроектировать некоторое изделие в режиме твердотельного проектирования.

- Затем, выбрав мой кастомный workbench пользователь может добавить в дерево модели процесс ХТО.

- Пользователь может всячески корректировать базовый процесс обработки (добавляя, удаляя этапы, редактируя их параметры).

- Создав предварительный план ХТО, пользователь может его провалидировать на корректность нажатием одной кнопки (плагин должен автоматически подтягивать параметры модели, такие как размеры, кривизна, материал и другие).

- Валидация должна выдать осмысленный результат, такой как: 
  1) Процесс ХТО валиден!;
  2) Процесс ХТО содержит одну или более неточностей: <далее список ошибок и, возможно, предложения по исправлению>.

- После того, как пользователь пройдёт некоторое количество итераций валидации он может запустить моделирование ХТО (нажав на соответствующую кнопку в интерфейсе (уже существующем туллбаре) плагина).

- Результатом моделирования процесса ХТО будет являться цветовая подсветка в реальном времени прямо на 3D-модели. Идея: модель подсвечивается цветами:
  1) зелёный — участок соответствует требованиям;
  2) жёлтый — участок с повышенным риском;
  3) красный — участок с критическим нарушением;
  4) синий/фиолетовый — зона недостаточной цементации;
  5) оранжевый — зона риска перегрева или коробления.

- После моделирования пользователь может нажать на кнопку отчёт и получить соответствующий отчёт по проведённому моделированию, включающий в себя следующее:
  1) Окно с пересчитанными параметрами изделия. Список параметров:
    1.1) Расчётная глубина цементированного слоя, м;
    1.2) Целевая глубина слоя, м;
    1.3) Поверхностная твёрдость после закалки, HRC (показатель твердости по шкале Роквелла (шкала C), выраженный в безразмерных условных единицах);
    1.4) Твёрдость сердцевины, HRC;
    1.5) Максимальный риск коробления, %;
    1.6) Риск перегрева тонких участков, %;
    1.7) Риск недостаточной цементации отверстий/пазов, %;
    1.8) Итоговая оценка процесса, Enum (требуется корректировка <список подсказок> / не требуется корректировок / процесс несёт высокие риски для конструкционной целостности изделия <список рисков>).
  2) Графики после моделирования, а именно:
    2.1) Концентрация углерода по глубине слоя;
    2.2) Твёрдость по глубине слоя;
    2.3) Температурный график техпроцесса;
    2.4) Изменение расчётной глубины слоя по времени.

---

Теперь конкретно по каждому из элементов интерфейса плагина:
  - Тулбар [CHT Tools]. Он содержит в себе кнопки: "Создать процесс", "Добавить этап", "Запустить моделирование", "Валидировать", "Отчёт".
  - Кнопка "Создать процесс" добавляет новый объект с базовой загатовкой процесса ХТО в дерево модели. Каждый из этапов содержит набор редактируемых параметров, характерных для него в секции Data.
  - Кнопка "Добавить этап" добавляет один из доступных на выбор этапов ХТО в существующий процесс.
  - Кнопка "Валидировать" производит статическую валидацию составленного техпроцесса и выдаёт результат, описанный выше.
  - Кнопка "Запустить моделирование" производит моделирование процесса ХТО в реальном времен.
  - Кнопка "Отчёт" составляет отчёт по результатам моделирования.
