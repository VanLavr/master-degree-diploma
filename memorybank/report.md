# Функциональные требования к кнопке «Отчёт»

## 1. Назначение функции

Функция «Отчёт» предназначена для формирования пользовательского отчёта по результатам проведённого моделирования процесса химико-термической обработки изделия.

Отчёт должен содержать численные результаты моделирования, графики, итоговую оценку процесса, краткие рекомендации и сведения о зонах риска, выявленных при моделировании.

Функция «Отчёт» не должна выполнять повторное моделирование. Она должна использовать ранее сформированный и сохранённый результат моделирования.

---

## 2. Общая логика работы функции

После нажатия кнопки «Отчёт» плагин должен:

1. проверить наличие сохранённого результата моделирования;
2. получить данные последнего моделирования;
3. сформировать таблицу расчётных параметров изделия;
4. сформировать графики по результатам моделирования;
5. сформировать текстовую итоговую оценку процесса;
6. отобразить отчёт пользователю в отдельном окне;
7. предоставить возможность сохранить отчёт во внешний файл.

---

## 3. Входные данные для формирования отчёта

### FR-REP-01. Получение результата моделирования

Плагин должен использовать последний сохранённый объект результата моделирования.

Объект результата моделирования должен содержать:

* сведения о модели;
* сведения о материале;
* параметры технологического процесса ХТО;
* расчётную глубину цементированного слоя;
* целевую глубину цементированного слоя;
* расчётную поверхностную твёрдость;
* расчётную твёрдость сердцевины;
* показатели рисков;
* данные для построения графиков;
* итоговую оценку процесса;
* список зон риска;
* карту цветовой визуализации модели.

Если результат моделирования отсутствует, плагин должен вывести сообщение:

```text
Отчёт не может быть сформирован: отсутствуют результаты моделирования.
Сначала запустите моделирование процесса ХТО.
```

---

### FR-REP-02. Использование актуальных данных моделирования

Отчёт должен формироваться на основе последнего успешно завершённого моделирования.

Если после моделирования пользователь изменил модель или параметры процесса, плагин должен предупредить пользователя, что отчёт может быть неактуален.

Пример сообщения:

```text
После последнего моделирования параметры процесса были изменены.
Рекомендуется повторно запустить моделирование перед формированием отчёта.
```

---

## 4. Состав отчёта

### FR-REP-03. Отображение общих сведений

Отчёт должен содержать общую информацию о выполненном моделировании:

* дата и время формирования отчёта;
* имя модели;
* материал изделия;
* тип процесса ХТО;
* список этапов технологического процесса;
* дата и время запуска моделирования;
* версия правил или конфигурации, использованной при расчёте, если такая информация хранится.

Пример:

```text
Модель: GearBody
Материал: 20Х
Процесс: цементация + диффузионная выдержка + закалка + отпуск
Дата моделирования: 12.06.2026 18:40
```

---

### FR-REP-04. Отображение параметров технологического процесса

Отчёт должен содержать таблицу исходных параметров ХТО, использованных при моделировании.

Для каждого этапа должны отображаться:

* название этапа;
* температура;
* длительность;
* технологическая среда;
* углеродный потенциал, если применимо;
* закалочная среда, если применимо;
* дополнительные параметры этапа, если они были заданы пользователем.

Пример таблицы:

| Этап       | Температура, °C | Длительность, мин | Дополнительные параметры |
| ---------- | --------------: | ----------------: | ------------------------ |
| Нагрев     |             850 |                40 | —                        |
| Цементация |             930 |               240 | Углеродный потенциал 1.1 |
| Закалка    |             830 |                20 | Масло                    |
| Отпуск     |             180 |                90 | —                        |

---

### FR-REP-05. Отображение пересчитанных параметров изделия

Отчёт должен содержать таблицу расчётных параметров изделия, полученных по результатам моделирования.

Минимальный набор параметров:

| Параметр                                        | Единица измерения         |
| ----------------------------------------------- | ------------------------- |
| Расчётная глубина цементированного слоя         | м                         |
| Целевая глубина цементированного слоя           | м                         |
| Поверхностная твёрдость после закалки           | HRC                       |
| Твёрдость сердцевины                            | HRC                       |
| Максимальный риск коробления                    | %                         |
| Риск перегрева тонких участков                  | %                         |
| Риск недостаточной цементации отверстий и пазов | %                         |
| Итоговая оценка процесса                        | Enum / текстовое значение |

При отображении глубины цементированного слоя допускается дополнительно выводить значение в миллиметрах для удобства пользователя.

Пример:

```text
Расчётная глубина слоя: 0.00085 м (0.85 мм)
Целевая глубина слоя: 0.00080–0.00100 м (0.80–1.00 мм)
```

---

### FR-REP-06. Формирование итоговой оценки процесса

Плагин должен формировать итоговую оценку процесса на основе результатов моделирования.

Допустимые значения итоговой оценки:

* `NO_CORRECTIONS_REQUIRED` — корректировки не требуются;
* `CORRECTIONS_REQUIRED` — требуется корректировка процесса;
* `HIGH_STRUCTURAL_RISK` — процесс несёт высокие риски для конструкционной целостности изделия.

Для пользовательского интерфейса значения должны отображаться в человекочитаемом виде:

```text
Корректировки не требуются.
```

```text
Требуется корректировка процесса:
- увеличить длительность цементации;
- проверить параметры закалки;
- снизить риск перегрева тонких участков.
```

```text
Процесс несёт высокие риски для конструкционной целостности изделия:
- высокий риск коробления тонких элементов;
- критический риск перегрева кромок;
- недостаточная цементация внутренних пазов.
```

---

### FR-REP-07. Отображение списка рекомендаций

Если моделирование выявило риски или отклонения, отчёт должен содержать список рекомендаций.

Рекомендации могут относиться к:

* изменению температуры цементации;
* увеличению или уменьшению длительности выдержки;
* изменению углеродного потенциала;
* изменению закалочной среды;
* добавлению или изменению этапа отпуска;
* изменению геометрии изделия;
* добавлению скруглений;
* увеличению толщины стенок;
* изменению требований к глубине слоя.

Пример:

```text
Рекомендации:
1. Увеличить длительность цементации для достижения целевой глубины слоя.
2. Добавить скругления на острых кромках для снижения риска перегрева.
3. Рассмотреть менее интенсивную закалочную среду для снижения риска коробления.
```

---

## 5. Графики отчёта

### FR-REP-08. График концентрации углерода по глубине слоя

Отчёт должен содержать график распределения концентрации углерода по глубине цементированного слоя.

Ось X:

```text
Глубина слоя, м
```

Ось Y:

```text
Концентрация углерода, %
```

График должен позволять оценить изменение содержания углерода от поверхности изделия к сердцевине.

---

### FR-REP-09. График твёрдости по глубине слоя

Отчёт должен содержать график распределения расчётной твёрдости по глубине слоя.

Ось X:

```text
Глубина слоя, м
```

Ось Y:

```text
Твёрдость, HRC
```

График должен показывать изменение твёрдости от поверхности изделия к сердцевине.

---

### FR-REP-10. Температурный график техпроцесса

Отчёт должен содержать температурный график технологического процесса ХТО.

Ось X:

```text
Время, мин
```

Ось Y:

```text
Температура, °C
```

На графике должны быть отражены основные этапы:

* нагрев;
* цементация;
* диффузионная выдержка;
* закалка;
* отпуск.

---

### FR-REP-11. График изменения расчётной глубины слоя по времени

Отчёт должен содержать график изменения расчётной глубины цементированного слоя во времени.

Ось X:

```text
Время, мин
```

Ось Y:

```text
Глубина цементированного слоя, м
```

График должен позволять оценить, в какой момент технологического процесса достигается требуемая глубина слоя.

---

## 6. Связь отчёта с визуальной моделью

### FR-REP-12. Отображение легенды цветовой карты

Отчёт должен содержать легенду цветовой карты, использованной при отображении результатов моделирования на 3D-модели.

Минимальная легенда:

| Цвет      | Значение                            |
| --------- | ----------------------------------- |
| Зелёный   | Участок соответствует требованиям   |
| Жёлтый    | Участок с повышенным риском         |
| Красный   | Участок с критическим нарушением    |
| Синий     | Зона недостаточной цементации       |
| Оранжевый | Зона риска перегрева или коробления |

---

### FR-REP-13. Отображение сводки зон риска

Отчёт должен содержать сводную таблицу зон риска, выявленных при моделировании.

Для каждой зоны риска должны отображаться:

* идентификатор зоны или элемента модели;
* тип риска;
* уровень риска, %;
* цветовая категория;
* краткая причина;
* рекомендация по исправлению.

Пример:

| Зона   | Тип риска                | Уровень риска | Категория | Причина       |
| ------ | ------------------------ | ------------: | --------- | ------------- |
| Face12 | Перегрев                 |          74 % | Оранжевая | Тонкая стенка |
| Face18 | Недостаточная цементация |          68 % | Синяя     | Узкий паз     |
| Edge5  | Критический риск         |          91 % | Красная   | Острая кромка |

---

### FR-REP-14. Добавление снимка модели с результатами моделирования

Плагин должен иметь возможность добавить в отчёт изображение 3D-модели с цветовой подсветкой результатов моделирования.

Снимок должен отражать текущее состояние визуализации во FreeCAD или сохранённую карту цветов результата моделирования.

Если снимок невозможно сформировать, отчёт должен быть сформирован без изображения с соответствующим предупреждением.

---

## 7. Отображение отчёта в интерфейсе

### FR-REP-15. Открытие окна отчёта

После нажатия кнопки «Отчёт» плагин должен открыть отдельное окно или панель предварительного просмотра отчёта.

Окно должно содержать:

* заголовок отчёта;
* общие сведения;
* таблицу исходных параметров процесса;
* таблицу расчётных параметров изделия;
* итоговую оценку;
* список рекомендаций;
* графики;
* сводку зон риска;
* легенду цветовой карты.

---

### FR-REP-16. Навигация по разделам отчёта

Окно отчёта должно позволять пользователю удобно просматривать разделы отчёта.

Допускается реализация через:

* вкладки;
* вертикальную прокрутку;
* дерево разделов;
* раскрывающиеся блоки.

Минимальный набор разделов:

* общие сведения;
* параметры процесса;
* результаты моделирования;
* графики;
* зоны риска;
* рекомендации.

---

### FR-REP-17. Обновление отчёта

Плагин должен позволять обновить содержимое отчёта на основе последнего сохранённого результата моделирования.

Обновление отчёта не должно запускать повторное моделирование.

---

## 8. Экспорт отчёта

### FR-REP-18. Сохранение отчёта во внешний файл

Плагин должен предоставлять возможность сохранить сформированный отчёт во внешний файл.

Рекомендуемые форматы:

* `.html`;
* `.pdf`;
* `.docx`.

Минимально допустимый формат для первой версии плагина — `.html`, так как он позволяет сохранить таблицы, текст, изображения и графики без сложной интеграции.

---

### FR-REP-19. Выбор пути сохранения отчёта

Перед экспортом отчёта плагин должен позволять пользователю выбрать путь сохранения файла.

Если пользователь отменяет выбор пути, экспорт должен быть отменён без ошибки.

---

### FR-REP-20. Экспорт графиков

При сохранении отчёта во внешний файл графики должны быть включены в отчёт в виде изображений или встроенных элементов.

Каждый график должен иметь:

* название;
* подписи осей;
* единицы измерения;
* читаемый масштаб.

---

### FR-REP-21. Экспорт снимка модели

Если в отчёт добавляется снимок модели, при экспорте он должен сохраняться вместе с отчётом.

Для HTML-отчёта допускается:

* встраивание изображения в файл;
* сохранение изображения рядом с HTML-файлом;
* использование относительного пути к изображению.

---

## 9. Обработка ошибок

### FR-REP-22. Ошибка отсутствия данных моделирования

Если данные моделирования отсутствуют, отчёт не должен формироваться.

Пользователю должно быть показано сообщение:

```text
Нет данных для формирования отчёта.
Сначала запустите моделирование процесса ХТО.
```

---

### FR-REP-23. Ошибка неполных данных моделирования

Если часть данных моделирования отсутствует, отчёт должен быть сформирован частично.

Отсутствующие значения должны отображаться как:

```text
Нет данных
```

или

```text
Расчёт не выполнен
```

При этом плагин должен показать предупреждение:

```text
Отчёт сформирован частично: часть результатов моделирования отсутствует.
```

---

### FR-REP-24. Ошибка построения графиков

Если один или несколько графиков не могут быть построены, отчёт должен быть сформирован без этих графиков.

В соответствующем разделе должно отображаться сообщение:

```text
График не может быть построен: отсутствуют исходные данные.
```

---

### FR-REP-25. Ошибка экспорта отчёта

Если отчёт не удалось сохранить во внешний файл, плагин должен вывести сообщение об ошибке с указанием причины.

Пример:

```text
Не удалось сохранить отчёт: выбранный каталог недоступен для записи.
```

---

## 10. Нефункциональные требования

### NFR-REP-01. Реализуемость в рамках FreeCAD Python Workbench

Формирование отчёта должно быть реализуемо средствами Python и FreeCAD.

Допускается использование:

* PySide для окна предварительного просмотра;
* matplotlib для построения графиков;
* стандартных средств Python для генерации HTML;
* дополнительных библиотек для экспорта в DOCX или PDF, если они доступны в окружении плагина.

---

### NFR-REP-02. Неразрушающий характер работы

Формирование отчёта не должно изменять геометрию модели, параметры процесса или результат моделирования.

---

### NFR-REP-03. Повторяемость отчёта

При одинаковых данных моделирования отчёт должен формироваться с одинаковыми численными значениями и одинаковой итоговой оценкой.

---

### NFR-REP-04. Читаемость отчёта

Отчёт должен быть понятен пользователю без необходимости анализа исходного кода плагина.

Все параметры должны иметь:

* название;
* значение;
* единицу измерения;
* краткое пояснение при необходимости.

---

### NFR-REP-05. Объяснимость результата

Итоговая оценка процесса должна сопровождаться пояснением.

Если процесс требует корректировки, отчёт должен показывать, какие параметры или зоны модели стали причиной такого вывода.

---

### NFR-REP-06. Производительность

Формирование отчёта по типовой модели должно выполняться за приемлемое время и не должно приводить к зависанию интерфейса FreeCAD.

Для длительного экспорта допускается отображение индикатора выполнения.

---

### NFR-REP-07. Отказоустойчивость

Ошибки построения отдельных разделов отчёта не должны приводить к аварийному завершению работы плагина.

Если отдельный раздел не может быть сформирован, отчёт должен быть сформирован без него с указанием причины.

---

### NFR-REP-08. Единообразие единиц измерения

Все численные значения в отчёте должны отображаться с единицами измерения.

Для малых величин допускается одновременный вывод в СИ и в инженерных единицах.

Пример:

```text
0.00085 м (0.85 мм)
```

---

### NFR-REP-09. Отделение отчёта от моделирования

Функция «Отчёт» должна использовать сохранённые результаты моделирования и не должна изменять результаты расчёта.

Повторный расчёт должен выполняться только через функцию «Запустить моделирование».

---

### NFR-REP-10. Возможность дальнейшего расширения

Архитектура формирования отчёта должна позволять в дальнейшем добавлять новые разделы без изменения основной логики плагина.

Например:

* сравнение нескольких вариантов ХТО;
* расширенные рекомендации;
