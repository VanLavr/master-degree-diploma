# Функциональные требования к кнопке «Добавить этап»

## 1. Назначение функции

Функция «Добавить этап» предназначена для добавления нового технологического этапа в ранее созданный процесс химико-термической обработки изделия.

Функция используется после создания процесса ХТО и позволяет пользователю расширить или уточнить технологический маршрут обработки.

Функция не выполняет валидацию, моделирование или формирование отчёта. Её задача — добавить в процесс новый редактируемый этап с начальными параметрами.

---

## 2. Общая логика работы функции

После нажатия кнопки «Добавить этап» плагин должен:

1. определить активный документ FreeCAD;
2. найти существующий процесс ХТО;
3. если процесс отсутствует — вывести ошибку;
4. отобразить пользователю список доступных этапов;
5. позволить пользователю выбрать тип добавляемого этапа;
6. создать новый этап внутри процесса;
7. заполнить параметры этапа значениями по умолчанию;
8. добавить этап в дерево процесса или внутреннюю структуру процесса;
9. обновить отображение процесса в интерфейсе;

---

## 3. Проверка исходных условий

### FR-AS-01. Проверка активного документа

Плагин должен проверять наличие активного документа FreeCAD.

Если активный документ отсутствует, этап не должен добавляться.

Сообщение пользователю:

```text
Невозможно добавить этап: отсутствует активный документ FreeCAD.
```

---

### FR-AS-02. Проверка наличия процесса ХТО

Плагин должен проверять, существует ли в текущем документе процесс ХТО.

Если процесс отсутствует, этап не должен добавляться.

Сообщение пользователю:

```text
Невозможно добавить этап: процесс ХТО ещё не создан.
Сначала создайте процесс ХТО.
```

---

### FR-AS-03. Проверка выбранного процесса

Если в документе существует несколько процессов ХТО, плагин должен определить, в какой процесс добавляется этап.

Допустимые варианты поведения:

* использовать выбранный пользователем процесс;
* использовать процесс, связанный с активной моделью;
* предложить пользователю выбрать процесс из списка.

Для первой версии плагина допускается использовать выбранный в дереве процесс ХТО.

---

## 4. Выбор типа этапа

### FR-AS-04. Отображение списка доступных этапов

Плагин должен отображать пользователю выпадающий список доступных типов этапов.

Минимальный список этапов:

* нагрев;
* цементация;
* диффузия;
* закалка;
* отпуск.

Дополнительно в будущем могут поддерживаться:

* охлаждение;
* повторный отпуск;
* нормализация;
* очистка поверхности;
* промежуточная выдержка.

---

### FR-AS-05. Загрузка списка этапов из конфигурации

Список доступных этапов должен быть задан в конфигурации плагина или в `rules.yaml`.

Рекомендуемый вариант — хранить список доступных этапов в `rules.yaml`.

Пример:

```yaml
available_stages:
  - id: "preheating"
    name: "Нагрев"
    allow_multiple: false

  - id: "carburizing"
    name: "Цементация"
    allow_multiple: false

  - id: "diffusion"
    name: "Диффузия"
    allow_multiple: true

  - id: "quenching"
    name: "Закалка"
    allow_multiple: false

  - id: "tempering"
    name: "Отпуск"
    allow_multiple: true
```

---

## 5. Создание нового этапа

### FR-AS-06. Создание объекта этапа

После выбора типа этапа плагин должен создать новый этап внутри текущего процесса ХТО.

Этап должен содержать:

* уникальный идентификатор;
* тип этапа;
* отображаемое имя;
* набор параметров;
* ссылку на родительский процесс.

Пример логической структуры:

```text
Процесс ХТО
  ├── Нагрев
  ├── Цементация
  ├── Диффузия
  ├── Закалка
  └── Отпуск
```

---

### FR-AS-07. Заполнение параметров этапа по умолчанию

Созданный этап должен автоматически получать набор параметров по умолчанию.

Примеры параметров:

Для этапа «Нагрев»:

* температура, °C;
* длительность, мин;
* скорость нагрева, если поддерживается.

Для этапа «Цементация»:

* температура, °C;
* длительность, мин;
* углеродный потенциал;
* технологическая среда;
* целевая глубина слоя, м.

Для этапа «Диффузия»:

* температура, °C;
* длительность, мин;
* среда обработки.

Для этапа «Закалка»:

* температура, °C;
* длительность выдержки, мин;
* закалочная среда;
* температура среды, если поддерживается.

Для этапа «Отпуск»:

* температура, °C;
* длительность, мин;
* количество циклов.

---

### FR-AS-08. Загрузка параметров этапа из rules.yaml

Параметры этапов по умолчанию должны загружаться из `rules.yaml`.

Пример:

```yaml
stage_defaults:
  diffusion:
    temperature_c: 880
    duration_min: 60
    atmosphere: "carburizing_gas"

  tempering:
    temperature_c: 180
    duration_min: 90
    cycles: 1
```

Если параметры не найдены в `rules.yaml`, плагин должен использовать встроенные базовые значения или вывести предупреждение.

---

### FR-AS-9. Возможность выбора позиции этапа

Плагин может позволять пользователю выбрать место вставки этапа.

Минимально допустимый вариант для первой версии — автоматическая вставка этапа в рекомендованное место согласно правилам процесса.

---

## 7. Отображение этапа в интерфейсе

### FR-AS-10. Отображение добавленного этапа в дереве FreeCAD

Новый этап должен отображаться в дереве модели как дочерний элемент процесса ХТО или в панели плагина как часть структуры процесса.

Пример:

```text
Процесс ХТО
  ├── 01_Нагрев
  ├── 02_Цементация
  ├── 03_Диффузия
  ├── 04_Закалка
  └── 05_Отпуск
```

---

### FR-AS-11. Автоматический выбор нового этапа

После добавления этапа плагин должен автоматически выбрать его в дереве или открыть его параметры в панели редактирования.

Это позволит пользователю сразу изменить параметры добавленного этапа.

---

### FR-AS-12. Сообщение об успешном добавлении

После успешного добавления этапа плагин должен вывести сообщение:

```text
Этап «Диффузия» успешно добавлен в процесс ХТО.
```

---

### FR-AS-13. Сброс актуальности предыдущих результатов

Если для процесса ранее были выполнены валидация, моделирование или отчёт, после добавления нового этапа плагин должен пометить эти результаты как устаревшие.

Физически удалять результаты не нужно, но они не должны считаться актуальными.

---

## 9. Сохранение данных

### FR-AS-14. Сохранение добавленного этапа в документе FreeCAD

Добавленный этап должен сохраняться вместе с документом FreeCAD.

После закрытия и повторного открытия файла структура процесса должна быть восстановлена.

---

## 10. Обработка ошибок

### FR-AS-15. Ошибка отсутствия процесса

Если процесс ХТО отсутствует, плагин должен вывести ошибку и не должен создавать отдельный этап вне процесса.

Сообщение:

```text
Этап не может быть добавлен: процесс ХТО не найден.
```

---

### FR-AS-16. Ошибка выбора этапа

Если пользователь не выбрал тип этапа из выпадающего списка, плагин должен отменить операцию без изменения процесса.

Сообщение:

```text
Тип этапа не выбран. Добавление этапа отменено.
```

---

### FR-AS-17. Ошибка создания этапа

Если этап не удалось создать, плагин должен вывести сообщение об ошибке.

Пример:

```text
Не удалось добавить этап: ошибка создания объекта этапа.
```

---

### FR-AS-18. Ошибка загрузки параметров по умолчанию

Если параметры этапа не удалось загрузить из конфигурации, плагин должен использовать встроенные значения или сообщить пользователю о проблеме.

Пример:

```text
Не удалось загрузить параметры этапа из rules.yaml. Использованы встроенные значения по умолчанию.
```

---

### FR-AS-19. Отмена операции без частичных изменений

Если добавление этапа завершилось ошибкой или было отменено пользователем, процесс не должен оставаться в частично изменённом состоянии.

---

# Нефункциональные требования к кнопке «Добавить этап»

## NFR-AS-01. Реализуемость в рамках FreeCAD Python Workbench

Функция должна быть реализуема средствами FreeCAD Python API и PySide без изменения исходного кода FreeCAD.

---

## NFR-AS-02. Неразрушающий характер работы

Добавление этапа не должно изменять геометрию CAD-модели.

Функция должна изменять только структуру процесса ХТО и связанные с ним данные.

---

## NFR-AS-03. Повторяемость поведения

При одинаковом типе процесса, одинаковом выбранном этапе и одинаковой конфигурации плагин должен создавать этап с одинаковой структурой и одинаковыми начальными параметрами.

---

## NFR-AS-04. Читаемость структуры процесса

Добавленный этап должен иметь понятное имя и корректное положение в структуре процесса.

Пользователь должен понимать назначение этапа без анализа внутреннего представления данных.

---

## NFR-AS-05. Сохраняемость данных

Добавленный этап и его параметры должны сохраняться в документе FreeCAD и восстанавливаться после повторного открытия файла.

---

## NFR-AS-06. Расширяемость

Архитектура добавления этапов должна позволять добавлять новые типы этапов без переработки основной логики функции.

Новые этапы должны подключаться через конфигурацию или отдельный реестр этапов.

---

## NFR-AS-07. Совместимость с последующими функциями

Добавленный этап должен быть доступен для дальнейшей работы функций:

* «Валидировать»;
* «Запустить моделирование»;
* «Отчёт».

---

## NFR-AS-08. Отказоустойчивость

Ошибка при добавлении этапа не должна приводить к аварийному завершению FreeCAD.

Плагин должен корректно обработать ошибку и показать пользователю понятное сообщение.

---

## NFR-AS-09. Производительность

Добавление этапа должно выполняться быстро и не должно приводить к заметному зависанию интерфейса FreeCAD.

---

## NFR-AS-10. Единообразие единиц измерения

Параметры добавленного этапа должны использовать единицы измерения, согласованные с остальными функциями плагина.

Рекомендуемые единицы:

* температура — °C;
* длительность — мин;
* глубина слоя — м с возможным отображением в мм;
* твёрдость — HRC;
* риск — %.

---

# Минимальный сценарий работы кнопки

```text
Пользователь нажимает «Добавить этап».
Плагин проверяет наличие активного документа.
Плагин проверяет наличие процесса ХТО.
Если процесс отсутствует — выводится ошибка.
Если процесс найден — пользователю отображается выпадающий список этапов.
Пользователь выбирает тип этапа.
Плагин создаёт новый этап внутри процесса.
Плагин заполняет параметры этапа значениями по умолчанию.
Плагин обновляет дерево процесса.
Плагин сообщает об успешном добавлении этапа.
```
